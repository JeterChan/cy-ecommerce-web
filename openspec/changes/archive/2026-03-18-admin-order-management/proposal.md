## Why

目前電子商務系統已具備基本的購物與結帳功能，但缺乏管理後台對訂單的有效監控與處理機制。管理員需要一個集中的介面來查看所有訂單、追蹤出貨狀態、處理客戶請求（如取消訂單）以及了解銷售情況，以確保營運順暢。

## What Changes

- **訂單列表頁面**：新增管理員專用的訂單概覽介面，支援依訂單編號、日期、狀態、客戶資訊進行篩選與搜尋。
- **訂單詳情頁面**：顯示訂單的完整資訊，包括訂購商品、收件人資訊、付款方式、物流追蹤及訂單紀錄。
- **訂單狀態管理**：提供管理員手動更新訂單狀態的功能（如：將狀態從「待出貨」改為「已出貨」）。
- **後端 API 增強**：
  - 新增訂單查詢 API（管理員權限）。
  - 新增訂單狀態更新 API。
  - 強化訂單資料模型以支援更完整的狀態流轉紀錄。

## Capabilities

### New Capabilities
- `admin-order-management`: 提供管理員完整的訂單生命週期管理能力，包含查詢、詳情檢視與狀態變更。

### Modified Capabilities
- `order-checkout-system`: 調整訂單狀態枚舉與模型，以支援管理後台所需的多樣化狀態（如：退貨中、已退款等）。

## Impact

- **API**: 新增 `/api/admin/orders` 系列端點，需嚴格的 RBAC (Role-Based Access Control) 驗證。
- **Frontend**: 在管理後台 (`/admin`) 新增訂單管理選單與對應的 Vue 組件。
- **Database**: `orders` 資料表可能需要新增 `updated_by`、`admin_note` 等欄位，並確保索引優化以利後台查詢。
