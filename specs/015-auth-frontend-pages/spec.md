# 功能規格書：驗證系統前端頁面擴展 (Auth Frontend Pages Extension)

**功能分支**: `015-auth-frontend-pages`  
**建立日期**: 2026-03-11  
**狀態**: 草稿  
**輸入**: 建立前端 Vue 個人檔案的頁面。新增頁面元件：1. 忘記密碼 2. 重設密碼 3. 個人檔案編輯 4. 刪除帳號按鈕 5. 信箱驗證頁面。需要正體中文。

## 使用者情境與測試 *(Mandatory)*

### 使用者情境 1 - 忘記與重設密碼 (Priority: P1)

當使用者忘記密碼時，可以透過輸入註冊信箱獲取重設連結，並在點擊連結後設定新密碼。

**為什麼設定此優先級**: 這是帳號安全管理的核心功能，確保使用者在丟失密碼時仍能找回帳號。

**獨立測試**: 可以透過進入忘記密碼頁面、輸入信箱、接收模擬郵件、進入重設密碼頁面並成功修改密碼來完整測試。

**驗收情境**:
1. **Given** 使用者在登入頁面點擊「忘記密碼」，**When** 輸入有效信箱並送出，**Then** 系統顯示已發送驗證信之提示。
2. **Given** 使用者點擊郵件中的重設連結，**When** 在重設頁面輸入新密碼並確認，**Then** 系統提示修改成功並導向登入頁。

---

### 使用者情境 2 - 個人檔案編輯與管理 (Priority: P1)

登入後的使用者可以在個人檔案頁面查看並編輯其個人資料。

**為什麼設定此優先級**: 個人化與資料正確性是會員系統的基本需求。

**獨立測試**: 進入個人檔案頁面，點擊編輯，修改欄位後儲存，重新整理頁面確認資料已更新。

**驗收情境**:
1. **Given** 使用者已登入，**When** 進入個人檔案頁面，**Then** 系統顯示目前的個人資訊。
2. **Given** 使用者點擊「編輯資料」，**When** 修改暱稱或電話並點擊儲存，**Then** 系統顯示儲存成功且內容已更新。

---

### 使用者情境 3 - 電子郵件驗證 (Priority: P1)

使用者註冊後點擊信箱中的驗證連結，自動完成帳號啟用。

**為什麼設定此優先級**: 確保使用者信箱的真實性，且為註冊流程的必要環節。

**獨立測試**: 使用帶有驗證 Token 的網址進入驗證頁面，確認頁面自動呼叫 API 並顯示結果。

**驗收情境**:
1. **Given** 使用者點擊驗證郵件連結，**When** 瀏覽器開啟驗證頁面，**Then** 系統自動呼叫後端驗證 API。
2. **Given** API 回傳成功，**When** 驗證完成，**Then** 頁面顯示驗證成功訊息並提供登入導向。

---

### 使用者情境 4 - 帳號刪除 (Priority: P2)

使用者決定不再使用服務時，可以選擇永久刪除其帳號。

**為什麼設定此優先級**: 符合隱私法規（如 GDPR）與使用者資料自主權。

**獨立測試**: 點擊刪除按鈕，觸發確認對話框，確認後執行刪除並強制登出。

**驗收情境**:
1. **Given** 使用者在個人檔案頁面，**When** 點擊「刪除帳號」，**Then** 系統彈出二次確認對話框。
2. **Given** 使用者確認刪除，**When** 處理完成，**Then** 系統清除登入狀態並導向首頁。

### 邊緣情況

- **連結過效**: 當使用者點擊已過期的密碼重設或信箱驗證連結時，系統應顯示清楚的錯誤提示。
- **重複驗證**: 若帳號已驗證過再次點擊連結，應提示「已完成驗證」。
- **網路中斷**: 在個人檔案儲存或刪除過程中網路斷線，應有重試機制或錯誤反饋。

## 需求 *(Mandatory)*

### 功能需求

- **FR-001**: 系統必須提供「忘記密碼」元件，包含電子郵件輸入欄位與格式檢查。
- **FR-002**: 系統必須提供「重設密碼」元件，包含新密碼輸入與再次確認輸入。
- **FR-003**: 個人檔案頁面必須顯示使用者資訊（如：Email、暱稱、註冊日期）。
- **FR-004**: 系統必須提供編輯個人資料的功能，允許修改可更動的欄位。
- **FR-005**: 個人檔案頁面必須包含「刪除帳號」按鈕，且點擊後需有防誤觸的確認對話框。
- **FR-006**: 系統必須建立獨立的「信箱驗證」落地頁面，該頁面掛載時需自動從網址參數取得 Token 並呼叫後端 `email-verify` API。
- **FR-007**: 所有 UI 文字、錯誤訊息與系統通知皆必須使用正體中文 (Traditional Chinese)。

### 關鍵實體 *(Key Entities)*

- **User Profile**: 代表使用者的個人資料，包含 Email、暱稱、電話、頭像等屬性。
- **Verification Token**: 用於信箱驗證與密碼重設的臨時憑證，具備時效性。

## 成功指標 *(Mandatory)*

### 衡量成果

- **SC-001**: 使用者能在 30 秒內完成「忘記密碼」申請流程。
- **SC-002**: 使用者點擊信箱驗證連結後，頁面應在 3 秒內顯示驗證結果。
- **SC-003**: 個人檔案編輯的儲存成功率應達到 99% 以上。
- **SC-004**: 所有表單驗證錯誤訊息均以正體中文顯示，且對非技術使用者清晰易懂。
