# 功能規格: 使用者個人檔案管理 API (User Profile Management API)

**功能分支**: `013-user-profile-api`
**建立日期**: 2026年2月23日 星期一
**狀態**: 草稿
**輸入**: 使用者描述: "建立auth模組的修改個人檔案相關API, 提供個人資訊的 CRUD API。"

## 使用者情境與測試 *(必填)*

### 使用者故事 1 - 檢視個人檔案 (優先級: P1)

身為一個已登入的使用者，我想要查看我的個人檔案資訊，以便確認我的帳戶詳情。

**為何此優先級**: 這是管理個人資料的基礎功能。

**獨立測試**: 使用已驗證的 Token 呼叫 API，檢查回傳的 JSON 是否包含正確的使用者資訊。

**驗收情境**:

1. **情境**：已登入使用者，**當**呼叫「取得個人檔案」API，**則**系統回傳包含使用者名稱、電子郵件、電話等資訊的 JSON 回應 (HTTP 200)。
2. **情境**：未登入使用者 (無 Token 或 Token 無效)，**當**呼叫此 API，**則**系統拒絕存取並回傳 401 Unauthorized。

---

### 使用者故事 2 - 更新個人檔案 (優先級: P1)

身為一個已登入的使用者，我想要修改我的個人資料 (如姓名、電話)，以便保持資訊更新。

**為何此優先級**: 允許使用者維護其資料的準確性。

**獨立測試**: 呼叫 API 更新特定欄位，然後再次呼叫「檢視個人檔案」API 確認變更已生效。

**驗收情境**:

1. **情境**：已登入使用者，**當**提交有效的更新資料 (如新暱稱)，**則**系統更新資料庫記錄並回傳更新後的使用者資訊 (HTTP 200)。
2. **情境**：使用者，**當**提交無效格式的資料 (如過長的姓名或格式錯誤的電話)，**則**系統拒絕更新並回傳 422 Unprocessable Entity 及錯誤詳情。
3. **情境**：使用者，**當**修改電子郵件地址時，**則**系統發送驗證郵件至新舊郵箱，並在兩者都驗證成功後才更新電子郵件。

---

### 使用者故事 3 - 刪除帳戶 (優先級: P2)

身為一個已登入的使用者，我想要刪除我的帳戶，以便移除我的個人資料。

**為何此優先級**: 滿足使用者對資料控制權的需求 (GDPR/隱私權考量)。

**獨立測試**: 呼叫刪除 API，確認成功後，嘗試再次登入應失敗。

**驗收情境**:

1. **情境**：已登入使用者，**當**請求刪除帳戶，**則**系統首先將帳戶標記為停用 (軟刪除)，並在 30 天後執行硬刪除。`並使目前的 Token 失效 (HTTP 200/204)。

## 假設

- 系統已存在 JWT 驗證機制 (`006-user-auth`)。
- 使用者 ID 可從 JWT Token 中解析獲得。
- 頭像 (Avatar) 目前僅儲存 URL，圖片上傳功能可能由其他模組處理 (或暫時手動輸入 URL)。
- 帳戶在被軟刪除後，將在 30 天後被硬刪除。

## 需求 *(必填)*

### 功能需求

- **FR-001**: 系統**必須**提供 `GET` API 讓已驗證使用者取得自己的個人檔案資訊。
- **FR-002**: 系統**必須**提供 `PATCH` (或 `PUT`) API 讓已驗證使用者更新其可編輯的個人資料欄位。
- **FR-003**: 系統**必須**驗證所有輸入資料的格式與長度限制。
- **FR-004**: 系統**必須**提供 `DELETE` API 讓已驗證使用者刪除或停用其帳戶。
- **FR-005**: 所有個人檔案 API **必須**要求有效的 Authentication Token (JWT)。
- **FR-006**: 修改密碼功能**必須**透過獨立的 API 或流程處理，不屬於個人檔案更新 API 的範疇。

### 關鍵實體 *(如果功能涉及資料則包含)*

- **UserProfile**: 包含使用者的可編輯屬性 (例如：Display Name, Phone Number, Address, Avatar URL)。
- **UserAccount**: 核心帳戶資訊 (Email, Password Hash, Status, Created At)。

## 假設

- 系統已存在 JWT 驗證機制 (`006-user-auth`)。
- 使用者 ID 可從 JWT Token 中解析獲得。
- 頭像 (Avatar) 目前僅儲存 URL，圖片上傳功能可能由其他模組處理 (或暫時手動輸入 URL)。

## 成功標準 *(必填)*

### 可衡量成果

- **SC-001**: 讀取個人檔案 API 的回應時間在 95% 的情況下小於 200ms。
- **SC-002**: 更新個人檔案 API 能正確處理所有定義的欄位驗證，並在資料無效時回傳明確錯誤訊息。
- **SC-003**: 帳戶刪除後，使用者無法再使用舊憑證登入。
