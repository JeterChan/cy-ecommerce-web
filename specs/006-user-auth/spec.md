# 功能規格: 使用者驗證

**功能分支**: `006-user-auth`  
**建立日期**: 2026年1月7日 星期三  
**狀態**: 草稿  
**輸入**: 使用者描述: "實作 User Domain & Repository. 整合 JWT Authentication, 實作登入/註冊功能. 並對密碼加密。提供"記住我” 的功能，讓使用者可以不必每次都登入。"

## 使用者情境與測試 *(必填)*

### 使用者故事 1 - 使用者註冊 (優先級: P1)

身為一個新使用者，我想要建立一個帳戶，以便存取個人化功能。

**為何此優先級**: 使用者領域的核心入口點；沒有註冊，就沒有使用者。

**獨立測試**: 可以透過嘗試使用新的電子郵件註冊並驗證帳戶是否已建立來完整測試。

**驗收情境**:

1. **情境**：訪客在註冊頁面，**當**他們輸入有效的電子郵件和密碼並提交，**則**會建立一個新的使用者帳戶，且他們會被登入或重定向到登入頁面。
2. **情境**：訪客，**當**他們嘗試使用已存在的電子郵件註冊，**則**系統會顯示錯誤訊息，指示帳戶已存在。
3. **情境**：訪客，**當**他們輸入無效的電子郵件或弱密碼，**則**系統會顯示驗證錯誤。

---

### 使用者故事 2 - 使用者登入 (優先級: P1)

身為一個註冊使用者，我想要使用我的憑證登入，以便存取我的帳戶。

**為何此優先級**: 安全存取系統的必要條件。

**獨立測試**: 可以透過使用已知有效帳戶登入並檢查成功的驗證回應（例如 JWT 權杖）來完整測試。

**驗收情境**:

11. **情境**：註冊使用者，**當**他們輸入正確的電子郵件和密碼，**則**他們會成功通過驗證並收到存取權杖 (JWT)。
12. **情境**：使用者，**當**他們輸入不正確的密碼，**則**系統會拒絕存取並顯示錯誤。
13. **情境**：使用者，**當**他們輸入不存在的電子郵件，**則**系統會拒絕存取（出於安全考量可選擇顯示通用錯誤）。

---

### 使用者故事 3 - 記住我 (優先級: P2)

身為一個使用者，我希望系統能記住我的登入會話，這樣我就不必每次造訪時都輸入密碼。

**為何此優先級**: 透過減少回訪使用者的操作摩擦來改善使用者體驗。

**獨立測試**: 可以透過勾選「記住我」選項登入，重新啟動客戶端/瀏覽器會話，並驗證使用者仍然保持登入狀態來測試。

**驗收情境**:

1. **情境**：使用者登入時，**當**他們勾選「記住我」選項並提交，**則**他們的會話在瀏覽器重新啟動後仍保持有效一段延長時間（例如 30 天）。
2. **情境**：使用者登入時，**當**他們沒有勾選「記住我」，**則**他們的會話在瀏覽器關閉時或在短暫的標準逾時後過期。

---

### 邊緣案例

- JWT 權杖過期時會發生什麼事？使用者應該會被提示再次登入。
- 系統如何處理密碼編碼？它必須正確處理特殊字元。
- 如果註冊期間資料庫關閉會發生什麼事？使用者應該看到友善的錯誤訊息。

## 需求 *(必填)*

### 功能需求

- **FR-001**: 系統**必須**允許使用者使用獨特的電子郵件地址和密碼註冊。
- **FR-002**: 系統**必須**驗證電子郵件的格式是否正確。
- **FR-003**: 系統**必須**在將密碼儲存到儲存庫之前，安全地雜湊/加密它們。
- **FR-004**: 系統**必須**透過驗證提供的憑證與儲存的資料來驗證使用者。
- **FR-005**: 系統**必須**在成功驗證後生成並返回一個 JSON Web Token (JWT)。
- **FR-006**: 系統**必須**支援「記住我」功能，以延長有效的會話持續時間。
- **FR-007**: 系統**必須**提供一個使用者儲存庫介面，用於資料持久化操作。
- **FR-008**: 系統**必須**允許使用者登出，使客戶端會話狀態失效。

### 關鍵實體 *(如果功能涉及資料則包含)*

- **使用者 (User)**: 代表使用系統的人的身份。屬性：ID、電子郵件、密碼雜湊、建立日期、更新日期。
- **驗證權杖 (AuthToken)**: 代表驗證證明 (JWT)。包含：主體 (UserID)、過期時間、發行者。

## 假設

- 密碼長度必須至少為 8 個字元。
- 「記住我」會話持續 30 天。
- 初始 MVP 註冊不要求電子郵件驗證（例如，透過連結）；允許立即登入。

## 成功標準 *(必填)*

### 可衡量成果

- **SC-001**: 使用者可以在 2 分鐘內完成註冊流程。
- **SC-002**: 95% 的登入 API 請求回應時間在 500 毫秒以內。
- **SC-003**: 100% 的儲存密碼都經過雜湊處理（無明文密碼）。
- **SC-004**: 「記住我」功能在瀏覽器重新啟動後能成功恢復使用者會話，無需手動登入。
